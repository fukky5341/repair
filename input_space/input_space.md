# Dividing Input Space

**Final Goal**: divide the input space into two sets:
- Positive set: input regions where all inputs in it satisfy the specification
- Negative set: input regions that contain buggy inputs that violate the specification

## Counterexample Generation in ProRepair
We follow the counterexample generation strategy in ProRepair.

**Goal**: identify a counter example that is the most likely to violate the specification

$$
x_\phi
=
\arg\max_{x \in \phi_{in}}
\sum_{\psi \in \phi_{cons}}
-\min\!\big(c_\psi^\top f(x) + d_\psi,\; 0\big).
$$

Exactly solving this optimization problem is difficult, so we use an approximate method.
1. Define the property input region $\phi_{in}$, usually as a box
2. Encode the the desired output as linear constraints $c_\psi^\top f(x) + d_\psi \ge 0$
3. For each $\psi$, compute a sound affine lower bound $w_\psi^\top x + b_\psi$ on $\phi_{in}$ using a verifier / LiRPA-style bound propagation.
4. Keep only constraints with negative lower bound, i.e. those in
$Vio(\phi)=\{\psi \mid \min_{x\in\phi_{in}} w_\psi^\top x + b_\psi < 0\}$.
5. Solve
    $$
    x_\phi \approx \arg\max_{x\in\phi_{in}}
    \sum_{\psi\in Vio(\phi)}
    -\big(w_\psi^\top x + b_\psi\big).
    $$


## Implementation
- Input space is derived from APRNN
    - e.g., for MNIST, buggy inputs are generated by adding fog corruption to the original training data, and the buggy input space is given by adding the perturbation
- Bound computation is done using auto-LiRPA

### Assumptions
- The number of v-polytopes is $n$
- The target label is $t$, so the violated constraints are of the form $f_i(x) - f_t(x) \le 0$ for $i \neq t$.

### Procedure
1. Input space ($\phi_{in}$)
2. Sound and Fast approximate bound computation <-- auto-LiRPA
    - check if the lower bound is negative for each constraint, and keep only those constraints
    - solve the optimization problem to get the counterexample candidates
    - check if the counterexample candidates actually violate the specification
        - if not (false positive): add the pool to further analyze
        - if yes (true positive): add the region that contains the counterexample to the pool for repair
3. Complete counterexample generation for perfectly dividing the input space into positive and negative sets
    1. split the input space into smaller regions
    2. compute the lower bounds for each region
    3. check if the lower bounds are negative for each region, and keep only those
    4. solve the optimization problem to get the counterexample candidates for each region
    5. check if the counterexample candidates actually violate the specification and add them to the positive/negative/undecided set accordingly
4. Iterate until all regions are classified as positive or negative
